# 某MMORPGログ分析プロジェクト

本プロジェクトの目的は、某MMORPGにおけるキャラクタ毎の所持枠拡張クエストのクリア状況を一元化して可視化することである。
上記目的を達成するため、クライアントアプリケーションが生成するログファイルを解析するプログラムを作成する。

## 背景知識および用語の定義

### 所持枠

某MMORPGにおいては、キャラクタが所持できるアイテムの数には上限がある。
以降、キャラクタが所持できるアイテムの数を所持枠と表記する。

### 所持枠拡張クエスト

所持枠はゲーム開始時点で12枠ある。
ただし、所持枠はゲーム内クエストをクリアすることで増加する。
以降、所持枠を増加させることができるゲーム内クエストを所持枠拡張クエストと表記する。

### 所持枠拡張クエストのクリア状況の確認方法

ゲーム内で特定のNPCに話しかけることで、
所持枠拡張クエストのクリア状況をまとめて確認することができる。

確認状況はゲーム内のメッセージウインドウに表示される。
また、クライアントアプリケーションの設定を変更することでログファイルへの出力を行うこともできる。
そのため、クライアントアプリケーションを起動していない場合であっても、ログファイルを確認することで所持枠拡張クエストのクリア状況を確認することができる。

ただし、クライアントアプリケーションの仕様によって、ログファイルはキャラクタ1体につき1つ作成される。そのため、複数キャラクタについて所持枠拡張クエストのクリア状況を確認したい場合は、
キャラクタ数と同じ数のログファイルを解析する必要がある。
